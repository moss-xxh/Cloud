# Cloud Drive 视觉规范与对齐原则

> 目标：云盘不是营销页，也不是后台工具。视觉方向应是“清爽、稳定、可信、长期耐看”的生产力工具。

## 1. 产品定位原则

- 类型：私有云盘 / 文件管理工具
- 核心任务：快速找到文件、上传下载、分享、管理文件
- 视觉关键词：清爽、克制、可靠、高效、轻量高级
- 禁止方向：炫技渐变、过度毛玻璃、过重阴影、营销页式动效、牺牲信息密度

参考顺序：
1. Google Drive：文件管理效率、信息层级、克制视觉
2. Notion：留白、轻表面、低干扰
3. Linear：精致边界、微阴影、状态反馈

## 2. 信息层级原则

页面层级必须稳定：

1. 当前路径 / 当前页面标题
2. 主操作：新建、上传、搜索
3. 文件内容区
4. 次级信息：大小、时间、类型、存储容量
5. 辅助操作：更多、排序、视图切换

约束：
- 文件内容区永远是主角，装饰不能抢文件内容。
- 顶栏和侧栏只提供导航和操作，不应制造过强视觉竞争。
- 选中态必须比 hover 态更明确。

## 3. 色彩原则

主色：蓝色，只用于可交互和关键状态。

推荐语义：
- primary：#1a73e8
- primary-hover：#1557d6
- surface：#ffffff / rgba(255,255,255,0.75)
- page-bg：#f5f7fb 到 #fafbff 的微渐变
- text-primary：#172033
- text-secondary：#667085
- text-muted：#7b8496
- border-subtle：rgba(255,255,255,0.75) 或 #e6edf7

文件类型色只用于缩略图/小图标，不扩散到全页面：
- 文件夹：蓝
- PDF：红
- 视频：紫
- 代码：绿
- Office：蓝
- 其他：靛蓝/灰蓝

约束：
- 单屏强色面积不能过多。
- 渐变只能用于主按钮、Logo、存储条或极轻背景。
- 不要让每个卡片都像独立广告牌。

## 4. 圆角原则

统一半径，不按感觉乱写：

- 小控件：10-12px
- 输入框 / 普通按钮：14-16px
- 文件卡片：16px
- 面板 / 登录卡片 / 空状态：24-32px
- 圆形头像/图标按钮：999px

约束：
- 同级组件圆角必须一致。
- 文件卡片不应比登录卡片更夸张。

## 5. 阴影与表面原则

阴影用于表达层级，不用于“显得高级”。

推荐：
- 默认卡片：0 8px 28px rgba(31,45,75,0.06)
- 悬浮卡片：0 14px 36px rgba(31,45,75,0.10)
- 登录卡片：0 24px 80px rgba(30,55,105,0.12)

约束：
- 主界面阴影必须轻，不能全页面漂浮。
- 毛玻璃只用于顶栏/侧栏/少量 surface，不要滥用。
- 文件卡片 hover 允许轻微上浮，但 scale 不超过 1.015；大量文件场景要稳。

## 6. 字体与字号原则

字体栈：
- Inter, SF Pro Display, -apple-system, BlinkMacSystemFont, Segoe UI, sans-serif

字号建议：
- 登录标题：32-36px / 600
- 页面标题：18-22px / 600
- 文件名：13-14px / 500-600
- 次级信息：11-12px / 500
- 按钮：13-14px / 600

约束：
- 中文文件名优先可读，不要为了精致压太小。
- 次级信息不能低到不可读。
- 文件名截断必须稳定，不能撑破卡片。

## 7. 密度原则

云盘是工具，不是展示页。

- 桌面网格：卡片宽度 160-200px 合理
- gap：16-20px
- 列表视图：应比网格更高效，不能过度稀疏
- 顶部区域：不能占用过多垂直空间

约束：
- 登录页可以“漂亮”；主列表必须“高效”。
- 如果视觉效果与查找效率冲突，优先效率。

## 8. 动效原则

动效只服务反馈：

- hover：轻微边框/阴影变化
- active/selected：明确状态变化
- 页面不做无意义动画
- transform scale ≤ 1.015
- transition 150-220ms

约束：
- 不做大幅漂浮、弹跳、强缩放。
- 文件卡片很多时，动效要稳，不要晃。

## 9. 可访问性原则

必须保证：

- 文本对比度足够
- focus-visible 清晰
- 按钮点击区域 ≥ 32px，主按钮 ≥ 40px
- 颜色不能是唯一状态信息，选中态还需要边框/背景/线条
- 移动端不能依赖 hover 才显示关键操作

## 10. 移动端原则

移动端不是桌面缩小版。

- 侧栏应为抽屉
- 搜索框不能挤压标题和头像
- 文件卡片建议 2 列，窄屏 1-2 列自适应
- 批量操作应考虑底部操作条
- 弹窗宽度和按钮高度要适配手指点击

## 11. 对当前 B+ 样板的修正判断

当前样板达成：
- 登录页从扁平灰底升级为现代登录页
- 主按钮、搜索框、文件卡片、侧栏选中态有明显改善
- 文件类型色彩开始建立认知

当前样板不足：
- 先做了视觉效果，再补原则，顺序错误
- token 未系统化，仍有大量 arbitrary class
- 毛玻璃、阴影、渐变需要收敛，避免全页面“漂”
- 移动端未完整验收
- 列表视图效率和信息密度还没按规范专项优化

## 12. 下一轮执行顺序

1. 先把本文件转成真实 design tokens / CSS variables
2. 收敛当前 4 个文件中过散的颜色、阴影、圆角
3. 用同一套 token 铺到共享、星标、回收站、搜索页
4. 做移动端专项验收
5. 最后再决定是否保留/减弱毛玻璃和 hover scale
